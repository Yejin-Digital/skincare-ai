# Skincare AI — Project Brief

> 이 문서는 "왜 이 프로젝트를 만드는가"에 대한 답입니다.
> 모든 디자인/기술 결정은 이 문서의 내용과 일치해야 합니다.

---

## 1. Problem Statement

스킨케어 제품을 살 때 사용자는 다음과 같은 문제를 겪는다:

- 제품 성분표를 봐도 무슨 뜻인지 모른다
- 자기 피부 타입과 고민에 맞는 성분이 무엇인지 찾기 어렵다
- 인플루언서나 광고에 의존하다 실패 구매가 반복된다
- 여러 제품을 같이 쓸 때 어떤 조합이 안전한지 모른다

결과적으로 **시간 낭비, 비용 손실, 피부 트러블**로 이어진다.

---

## 2. Target User

### Primary Persona: "성분 초보 효율러" (Jina, 27)
- 20대 후반 직장인
- 피부 고민: 민감성 + 가끔 여드름 + T존 유분
- 화장품을 사기 전 리뷰를 1시간 이상 찾아본다
- 성분 이름을 들어봤지만 정확히 뭔지는 모른다
- 가성비 중시, 1~5만원대 제품을 주로 산다
- 디바이스: 주로 스마트폰, 가끔 노트북

### Secondary Persona: "성분 입문 탐험가" (Min, 22)
- 대학생
- 피부 고민: 여드름성, 모공
- SNS에서 화장품 정보를 자주 본다
- 첫 스킨케어 루틴을 만드는 중
- 디바이스: 거의 100% 스마트폰

---

## 3. Core Value Proposition

> "내 피부 조건을 입력하면, AI가 추천하는 성분과 그 성분이 들어있는
> 실제 구매 가능한 제품을 한 번에 보여준다."

기존 서비스(화해, 글로우픽 등)와의 차별점:
- **성분 → 제품**의 흐름 (기존: 제품 → 성분)
- **AI 분석 근거**를 함께 제공해서 신뢰감 형성
- **조합 추천 + 주의 성분** 정보로 루틴 설계까지 도움

---

## 4. Core User Flow (MVP)

```
Landing
  ↓ [CTA: 내 피부 진단하기]
Diagnosis Flow (4 steps)
  ├─ Step 1: 기본 정보 (나이대, 성별)
  ├─ Step 2: 피부 타입 (건성/지성/복합/민감)
  ├─ Step 3: 피부 고민 (멀티 선택)
  └─ Step 4: 라이프스타일 (선택 사항)
  ↓ [분석 중... AI 호출]
Results
  ├─ 추천 성분 카드 (3~5개, 각 성분의 효능 + AI 분석 근거)
  ├─ 성분별 추천 제품 리스트
  └─ 각 제품의 구매처(온/오프라인) + 가격대
  ↓ [제품 클릭]
Product Detail
  ├─ 제품 상세 정보
  ├─ 함께 쓰면 좋은 제품 조합
  └─ 주의해야 할 성분 조합
```

## 5. Success Metrics (포트폴리오 평가 기준)

이 프로젝트가 "잘 됐다"의 기준:

1. **완성도**: 진단 → 결과까지 끊김 없이 동작
2. **반응형**: 모바일/태블릿/데스크탑 모두 자연스러움
3. **AI 활용**: AI가 단순 키워드 매칭이 아닌 실제 추론을 보여줌
4. **디자인 일관성**: 디자인 시스템이 모든 페이지에서 일관되게 적용
5. **신뢰감**: 추천 근거가 명확해서 "왜?"라는 질문에 답이 됨

## 6. Scope

### ✅ In Scope (MVP)
- 4단계 진단 플로우
- AI 기반 성분 추천
- 제품 리스트 + 구매처/가격 정보
- 제품 상세 + 조합 추천
- 완전 반응형 (mobile / tablet / desktop)
- 다크모드 지원

### ❌ Out of Scope
- 사용자 계정 / 로그인
- 결제 / 실제 구매 처리
- 리뷰 / 커뮤니티 기능
- 푸시 알림
- 다국어 (한국어만)
- 실시간 가격 크롤링 (시드 데이터 사용)

---

## 7. Tone & Manner

### Brand Voice
- **Trustworthy** — 과장 없음, 근거 제시
- **Clean** — 시각적으로도 언어적으로도 군더더기 없음
- **Personal** — "당신의 피부에 맞춰", 친근하지만 가볍지 않은

### Design Direction
- 의료/약국 느낌이 아닌 **현대적 라이프스타일 톤**
- 흰색/오프화이트 기반 + 1~2개의 액센트 컬러
- 사진보다 **타이포그래피와 여백** 중심
- 일러스트는 추상적/기하학적 (인종/피부톤 편향 방지)

---

## 8. Technical Constraints

- 모든 AI 호출은 서버사이드(Next.js API Route)에서 처리
- API 키는 환경변수로만 관리
- 첫 로딩 3초 이내 목표
- Lighthouse Performance 90+ 목표
- WCAG AA 접근성 준수
